* PLAIN TALK

这篇论文想解决的问题很朴素：让大语言模型在“需要多步思考”的题上别靠猜，而是真的把推理过程走完。作者发现，只要在提示里给几个“题目 -> 思考过程 -> 答案”的示例（Chain-of-Thought, CoT），大模型就会学着先写推理再给答案，准确率会大幅提升，尤其在算术、常识和符号推理任务上。

* NAPKIN FORMULA

#+begin_example
+---------------------------------------------------------------+
|                                                               |
|  最终答案 = 大模型(问题 + K 个示例(题 -> 思考链 -> 答案))     |
|                                                               |
+---------------------------------------------------------------+
#+end_example

含义：把 K 个带“思考链”(chain-of-thought，指可读的中间推理步骤)的示例拼进提示里，让大模型在新问题上也先生成中间推理，再输出最终答案。

* PROBLEM

*痛点定义*: 在不做微调的前提下，如何让语言模型在需要多步推理的任务上显著变准，并且把“它是怎么想的”以可读方式暴露出来。

*前人困境*:  
- 标准 prompting 往往要求模型“直接给答案”，模型更像在做模式匹配，容易跳过中间步骤，遇到长链推理就崩。  
- 用训练/微调让模型学会写推理（scratchpad/rationale）有效，但门槛高：要数据、要训练、要迭代。  

* INSIGHT

*核心直觉*: 大模型不是不会推理，而是“默认不把推理写出来”；只要用少量示例在提示里把“先写推理再答”的格式立起来，它就会把更多计算预算分配到中间步骤，从而提高正确率。

*关键步骤*:
1. *把示例答案改写为“思考链 + 最终答案”*: few-shot exemplars 不只给答案，还给可读的中间推理步骤，让模型在 in-context 学到输出结构。
2. *让模型在测试题上照着生成*: 对新问题，模型先生成多步推理文本，再在末尾给出最终答案（评测时可从末行/固定格式抽取）。
3. *验证不是“多写字”而是“写对步骤”*: 消融显示，单纯把输出变长或只写方程式不够，关键在于连贯的中间推理链条。

* DELTA

*vs SOTA*: 论文在多类推理任务上展示了显著提升（跨 GPT-3 175B、PaLM 540B、LaMDA 137B）。典型例子：  
- *算术推理 GSM8K*: GPT-3 175B 从 17.9% -> 46.9%，PaLM 540B 从 56.9% -> 80.7%，LaMDA 137B 从 15.7% -> 57.2%。  
- *符号推理 Last Letter Concatenation*: GPT-3 175B 从 0% -> 61.1%，PaLM 540B 从 14.8% -> 74.7%。  
- *常识/理解类 BIG-bench 子任务*: Date Understanding、Sports Understanding 等也有大幅提升（例如 PaLM 540B: 35.1% -> 70.8%，51.4% -> 70.9%）。  

*新拼图*:  
- 给出了一个“几乎零成本”的推理增强手段：不训练、不改参数，只改提示结构。  
- 明确展示了一个 scaling 现象：CoT 的收益随着模型规模增长而出现/放大（小模型很难稳定学到这件事）。  
- 同时带来一种弱可解释性接口：模型输出的中间步骤可被人类直接阅读、检查与调试。  

* CRITIQUE

*隐形假设*:
- *模型规模足够大*: 论文结果强依赖 100B+ 量级；对小模型，CoT 往往学不稳，甚至输出“看起来像推理”的废话。
- *示例思考链质量高且贴近任务分布*: CoT prompting 本质是 in-context 模仿；示例写法、步骤粒度、格式与任务相似性都会强影响效果。
- *“写出来的推理”≈“真实推理过程”*: 输出的思考链可能是事后编造的合理化文本（看着对，但并不保证内部计算真的按这条链走）。

*未解之谜*:
- 如何系统化/自动化地产生高质量思考链示例（减少人工标注与 prompt 工程成本）。
- 如何让小模型也稳定获益：能否通过蒸馏、训练目标或解码策略，把 CoT 能力迁移下去。
- 如何把“可读推理”变成“可验证推理”：让中间步骤可检查、可纠错、可约束，避免高置信幻觉。
